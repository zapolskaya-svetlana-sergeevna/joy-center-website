Живая ссылка на работающий проект: [joy-center.infinityfree.me](http://joy-center.infinityfree.me)

**J.O.Y. Wellness Space** — это комплексная веб-платформа для автоматизации работы психологического центра. Проект представляет собой монолитное веб-приложение, реализующее систему онлайн-записи к специалистам, групповую терапию с листами ожидания, блог с публикациями, а также интернет-магазин цифровых материалов (медитаций и курсов) с корзиной и обработкой заказов.

Ролевая модель и функционал личных кабинетов (RBAC)

Платформа имеет гибкое разграничение прав доступа (Role-Based Access Control) на основе сессий PHP для трех типов пользователей:

1. Личный кабинет Клиента 
* **Запись на сессии:** Выбор психолога по специализации, выбор свободного временного слота из динамической сетки расписания и оформление записи.
* **Групповая терапия:** Запись на воркшопы. Реализована логика **листа ожидания (Waitlist)**: если места закончились, клиент попадает в очередь. При отмене записи другим участником система автоматически переводит первого клиента из листа ожидания в активный статус.
* **Корзина и покупки (E-commerce):** Выбор и покупка цифровых продуктов (медитаций, вебинаров), управление корзиной в реальном времени, оформление заказа.
* **Антифрод система отзывов:** Оставить отзыв специалисту может только тот клиент, у которого есть хотя бы одна завершенная сессия (`status = 'completed'`) с данным психологом.

2. Личный кабинет Психолога
* **Управление расписанием:** Открытие свободных окон для записи, просмотр личного календаря сессий.
* **Ведение клиентов:** Управление статусами визитов (подтверждение, завершение, отмена).
* **Контент-маркетинг:** Создание и редактирование профессиональных статей для блога центра через встроенный WYSIWYG-редактор.

3. Панель Администратора (CRM)
* **Управление командой:** Добавление новых специалистов, заведение их учетных записей, редактирование анкет и стажа.
* **Обработка заказов:** Модерация покупок цифровых продуктов, ручное открытие доступа к материалам для клиентов после подтверждения оплаты.
* **Модерация отзывов:** Проверка отзывов клиентов перед публикацией на сайте (одобрение или отправка в спам).
* **Настройки ИТ-инфраструктуры:** Управление справочниками кабинетов (залов), тем обращений клиентов, редактирование контактов организации и FAQ.
* **Почтовые рассылки:** Функция массовой email-рассылки уведомлений клиентам через PHPMailer.


Технологический стек

* **Backend:** PHP (Vanilla PHP, архитектура без фреймворков для максимального понимания базовых протоколов и сессий).
* **Работа с БД:** MySQL, библиотека PDO (интеграция prepared statements для защиты от SQL-инъекций).
* **Почтовый сервис:** PHPMailer (интеграция SMTP для отправки писем о сбросе паролей и заказах).
* **Frontend:** HTML5, CSS3 (Bootstrap 4, FontAwesome 5), JavaScript (jQuery, библиотека IMask для валидации телефонных номеров).


Архитектура БД и безопасность

Проект использует реляционную базу данных. Дамп структуры БД прикреплен в корне репозитория (joy_db.sql).

Ключевые архитектурные решения:
1. **Транзакции базы данных (ACID):** Использование транзакционных блоков `beginTransaction()` и `rollBack()` в критических процессах (оформление заказа, запись на сессию), чтобы исключить рассинхронизацию таблиц при сбоях.
2. **Безопасность авторизации:** Безопасное хэширование паролей с помощью алгоритма `bcrypt` (`password_hash`), валидация длины пароля.
3. **Восстановление доступа:** Генерация уникальных хэш-токенов (`bin2hex(random_bytes(32))`) с ограничением времени действия (expires) для безопасного сброса пароля через email.




В рамках работы над проектом были составлены тест-кейсы и чек-листы для проверки критических путей приложения:
* **Негативные тесты безопасности:** Проверка защиты от несанкционированного доступа (IDOR) — попытка зайти в админ-панель без роли `admin` через прямую ссылку.
* **Тестирование логики корзины:** Граничные значения стоимости, поведение корзины при обновлении сессии, обработка пустой корзины.
* **Интеграционные тесты:** Проверка корректности записи связей в таблицы базы данных `group_participants` и `order_items` после отправки форм.
